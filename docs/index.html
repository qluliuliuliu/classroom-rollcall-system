基于有状态的课堂点名系统

面向对象程序设计(Java) 课程设计 · 团队项目博客

一、项目简介

本系统是一个面向高校课堂教学的辅助工具，旨在解决传统随机点名中"有的同学被点名多次、有的同学从未被点到"的不公平问题。 系统通过记录每位学生的历史被点名次数和回答问题次数，采用加权随机算法，确保每位学生获得均等的点名机会。 当连续多人无法回答问题时，自动启动救场机制，避免课堂僵局。

三、功能需求分析

学生管理

单个/批量添加学生、Excel导入导出、搜索过滤、请假/销假管理、数据持久化

智能点名

加权随机点名、自动救场机制、题目太难提示、继续提问模式、请假自动排除

数据统计

汇总卡片（总数/点名/答出/成功率）、排名表、点名频次分布

四、项目亮点

  1. 加权随机算法：权重 = 100/(被点名次数+1)，累积权重法选择，确保点名机会均等
  2. 双重智能救场机制：连续3人未答出 → 从高回答率学生中选取 → 救场3连败 → 提示"题目太难，建议老师讲解"
  3. 完整的请假管理：搜索标记请假学生，点名时自动排除缺席者
  4. MVC+DAO+策略模式：三层架构分离，面向接口编程，更换存储方式无需修改上层代码
